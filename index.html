Quem é, como atua e por que importa

Identidade

  • Especialista virtual em palco, luz, LED e orçamento.
  • Opera no ecossistema Stage AI (Telegram → OpenClaw → Stage AI Lite).
  • Persona definida em SOUL.md e AGENTS.md.

Postura

Técnica, direta e orientada a regras. Reforça limites críticos (LED, performers, fixtures) para garantir renderizações fiéis e propostas consistentes.

Conectar briefing A–F a render, orçamento e documentação visual/financeira pronta para cliente e time.

Capacidades

Núcleo Operacional

6 pilares que sustentam o agente

1. Render 4K / 30s

Converte briefing A–F em payload JSON auditado e chama Stage AI Lite com defaults (palco circular, câmera wide, semCenaHumana por padrão).

2. Variações

Altera LED, fixtures, arte ou performer sem refazer briefing; mantém histórico para controlar créditos e coerência visual.

3. Orçamento

Consulta pricing.csv e calcula item a item (palco, LED, iluminação, operadores, logística). Entrega resumo pronto para o cliente.

4. Consumo Energético

Estima potência pico/média, corrente e kWh com base em LED (m²/pixels) e fixtures, ajudando a prever infraestrutura.

5. Decks & Dossiês

/design_systems transforma Markdown em apresentações Stage AI (logo, fontes, grids) incluindo temas “Wiley Edition”.

6. Lighting Designer

/lighting_designer suggest analisa payloads, distribui FOH/laterais/back/teto, recomenda haze/CCT e gera patch JSON.

Stage AI Lite

Engine de Geração de Imagens

API proprietária que o agente chama em 30 s

Endpoint

POST ${STAGEAI_BASE_URL}/api/lite/generate com cabeçalho X-Lite-Key (stageai lite).

Recebe JSON completo com campos A–F, LED, luz, paleta, cena humana e notas.

Payload

  • Defaults auditados: palco circular, semCenaHumana=true, câmera wide.
  • Permite ledLayout, luzPosicoes, artePaleta, performance, cameraStyle.
  • Respeita regras SOUL (sem LED inventado, performer correto).

Resposta

JSON com image_url, remaining_credits e eco do payload.

Tempo médio: 30s, resolução 4K, estilo Stage AI.

Autenticação

Variáveis de ambiente: STAGEAI_BASE_URL e STAGEAI_LITE_API_KEY carregadas via ~/.profile.

Aprovação do gateway permite curl com POST.

Logs & Créditos

Resposta cita créditos restantes. AGENT loga payload + image_url em memória diária para /variacao.

Erros retornam status HTTP + mensagem; agente repassa ao usuário.

Integração com Skills

  • /render_quick: monta JSON e chama API.
  • /variacao: reaproveita payload e envia diffs.
  • /lighting_designer: injeta patch de luz antes da chamada.
Stage AI Lite roda em Vercel (stack proprietária Stage AI) — manter credenciais seguras.

Checklists & Acervo

Como replicar ou manter o sistema

Checklist de Setup

  • Instalar Node 22+ e OpenClaw no host (RPi4).
  • Rodar openclaw onboard --install-daemon.
  • Configurar env: OPENAI_API_KEY, TELEGRAM_BOT_TOKEN, STAGEAI_BASE_URL, STAGEAI_LITE_API_KEY.
  • Copiar workspace Stage AI (IDENTITY, SOUL, skills).
  • Aplicar approvals para curl/bash/python.
  • Testar Stage AI Lite com curl antes de liberar no bot.

Acervo Local

Local: /home/labnomade/stageai_acervo (~3 MB). Contém decks PDF/HTML, logotipo, fontes, pricing.csv, dossiês e pastas stageai_docs/ stageai_dossier/.

Monitoramento

Gateway approvals listam comandos autorizados (curl, python3, bash). Serviços monitorados via systemctl --user (openclaw-gateway).
